⚠️ WARNING: This component demonstrates outdated patterns. See the markdown guide for detailed explanations of what NOT to do.

Pet Details (ID: {{ petId() }}) - Manual State Management @if (isLoading()) {

Loading with manual state...

} @if (error(); as errorMsg) {
Error: {{ errorMsg }}
} @if (pet(); as petData) {

{{ petData.name }}

Type: {{ petData.type }}

Breed: {{ petData.breed }}

Age: {{ petData.age }} years

Owner: {{ petData.owner }}

{{ petData | json }}
}

Why This Pattern is Problematic:

  • ❌ Manual subscription management = memory leak risk
  • ❌ Must manually coordinate 3 signals (petId, pet, isLoading, error)
  • ❌ No automatic reactivity when signals change
  • ❌ Verbose .subscribe() syntax and error handling
  • ❌ Hard to test - lots of manual state coordination
  • ❌ Requires lifecycle hooks (OnInit, OnDestroy)

✅ Use resource() instead! It handles all of this automatically with reactive updates.
